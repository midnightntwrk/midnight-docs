# Tokens on Midnight

Midnight uses a dual-token model with two native tokens: NIGHT and DUST. NIGHT is transferable, used for staking and governance, and generates DUST over time. DUST is shielded, non-transferable, and pays transaction fees. Both operate across a dual ledger that supports shielded (private) and unshielded (public) state.

This section covers how tokens work on Midnight, from the native tokens to custom tokens you create in Compact.

## Native tokens

- **NIGHT**: the native utility token. You hold NIGHT for staking, governance, and DUST generation. Unit: STAR (1 NIGHT = 10^6 STAR).
- **DUST**: a shielded, non-transferable fee resource. Your NIGHT balance generates DUST over time up to a cap, and every transaction consumes it. Unit: SPECK (1 DUST = 10^15 SPECK).

For the economic rationale behind this design, see [Dual-component tokenomics](/concepts/dual-component-tokenomics). For the technical architecture of DUST (generation, decay, registration), see [DUST architecture](/concepts/dust-architecture).

## Shielded and unshielded

Midnight's dual ledger lets tokens exist in two forms:

- **Shielded**: private state where wallet addresses and transaction details stay confidential. 1AM and urble default to shielded.
- **Unshielded**: public state visible on the ledger, similar to transparent blockchains. Lace is opt-in for shielded.

Both NIGHT and custom tokens can move between shielded and unshielded state. For code examples of shielded and unshielded token transfers in Compact, see [Token transfers](/examples/contracts/token-transfers).

## Custom tokens

You can create your own fungible and non-fungible tokens in Compact using the standard library's token functions (`mintShieldedToken`, `sendShielded`, `receiveShielded`, and their unshielded counterparts). The [OpenZeppelin contracts for Compact](/sdks/community/openzeppelin-compact-contracts) library provides token standards, access control, and security primitives modeled on the Solidity originals.

## Fees

Every transaction on Midnight requires DUST. On a fresh wallet, DUST takes time to generate: about 12 hours on mainnet or testnet, and about 5 minutes on a local network. To get started:

- [Get faucet tokens](/guides/acquire-tokens) (tNIGHT and tDUST on Preprod)
- [Generate DUST programmatically](/guides/generating-dust-programmatically) (using the wallet SDK)
